## Why this matters

`adm-zip` is a direct runtime dependency of the published `walletwright` package, used to extract
downloaded wallet extension archives. Versions `<0.6.0` carry a HIGH advisory (GHSA-xcpc-8h2w-3j85):
a crafted ZIP triggers a roughly 4GB memory allocation. The manifest pins `^0.5.16`, and because
caret on a `0.x` version locks the minor, `pnpm install` can never resolve to the patched `0.6.0`.
Every consumer of the package inherits the vulnerable line until the manifest is bumped. Reachability
in normal use is low (archives come from trusted vendor URLs and `download.ts` already guards
zip-slip), but shipping a HIGH advisory in a published dependency is worth closing.

## Current state

- `packages/walletwright/src/internal/download.ts` is the only `adm-zip` consumer. Excerpt:

```ts
// download.ts:6
import AdmZip from "adm-zip";
// download.ts:66-75
const zip = new AdmZip(zipBytes);
const root = path.resolve(outDir);
for (const entry of zip.getEntries()) {
  const target = path.resolve(root, entry.entryName);
  if (target !== root && !target.startsWith(root + path.sep)) {
    throw new Error(`[walletwright] refusing to extract ${entry.entryName}: escapes ${outDir}`);
  }
}
zip.extractAllTo(outDir, /* overwrite */ true);
```

- `packages/walletwright/package.json` current pins:

```json
"dependencies": {
  "adm-zip": "^0.5.16",
  "classic-level": "^1.4.1"
},
"devDependencies": {
  "@types/adm-zip": "^0.5.7"
}
```

- The API used (`new AdmZip(buffer)`, `getEntries()`, `entry.entryName`, `extractAllTo`) is stable
  across 0.5 to 0.6; `0.6.0` is a maintenance release.
- Tests that must keep passing: `packages/walletwright/src/internal/download.test.ts` (extraction,
  zip-slip, and sha256 cases).

## Steps

### Step 1: Bump the dependency ranges

In `packages/walletwright/package.json`, change `"adm-zip": "^0.5.16"` to `"adm-zip": "^0.6.0"`. Then
run `pnpm view @types/adm-zip version`: if a `0.6.x` exists, bump `"@types/adm-zip"` to it; if the
latest is still `0.5.x`, leave it (the 0.5 types stay compatible with the 0.6 runtime).

**Verify**: `grep '"adm-zip"' packages/walletwright/package.json` shows `^0.6.0`.

### Step 2: Reinstall and regenerate the lockfile

Run `pnpm install`.

**Verify**: `pnpm audit --prod` no longer lists the `adm-zip` "Crafted ZIP file triggers 4GB memory
allocation" advisory. (Other transitive advisories may remain; out of scope here.)

### Step 3: Confirm extraction still works

**Verify**: `pnpm turbo run test --filter=walletwright` all pass (including every `download.test.ts`
case); `pnpm typecheck` exit 0; `pnpm --filter walletwright build` exit 0.
