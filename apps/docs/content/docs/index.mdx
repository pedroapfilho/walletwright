description: walletwright is Playwright wallet automation for MetaMask (EVM), Phantom (EVM + Solana), and Slush (Sui). Onboard once, then connect and sign in real browser extensions.
---

walletwright drives **real wallet extensions** in Playwright tests. It onboards a wallet from a seed
phrase into a cached browser profile, then unlocks it and clicks through the extension's connect and
signature popups against your dapp. There are no mocked providers and no forked test framework, just
`@playwright/test` and the actual MetaMask, Phantom, and Slush extensions.

The approach is simple and proven: onboard once, cache the profile, drive the popups. walletwright
runs it on plain `@playwright/test` with current wallet and Chromium versions, with no fork and no
version pinning to maintain.

## At a glance

- Real MetaMask, Phantom, and Slush builds, loaded unpacked into Chromium. No mocked providers.
- One `wallet` fixture covers EVM (`window.ethereum`), Solana (`window.phantom.solana`), and Sui (the
  Wallet Standard).
- `buildCache()` imports the seed into a profile on disk once. Tests launch from a copy and only
  unlock, so they start in seconds.
- `createWalletFixtures(setup)` returns a `@playwright/test` `test` with a `wallet` fixture, so there
  is no framework lock-in.

## A test, end to end

```ts
import { createWalletFixtures } from "walletwright";
import { metamask } from "./wallet-setup";

const test = createWalletFixtures(metamask);
const { expect } = test;

test("connect and sign", async ({ page, wallet }) => {
  await page.goto("/");

  await page.getByRole("button", { name: "Connect" }).click();
  await wallet.connectToDapp();
  await expect(page.locator("#account")).toContainText("0x");

  await page.getByRole("button", { name: "Sign" }).click();
  await wallet.confirmSignature();
});
```
