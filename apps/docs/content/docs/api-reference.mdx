Everything is exported from the package root:

```ts
import {
  buildCache,
  createWalletFixtures,
  launchWallet,
  wallets,
  walletKindsByEcosystem,
  type Ecosystem,
  type Wallet,
  type WalletSetup,
  type WalletKind,
  type WalletDefinition,
} from "walletwright";
```

## `WalletSetup`

The wallet to import and the credentials to unlock it.

```ts
type Ecosystem = "evm" | "svm" | "sui" | "dot" | "btc";
type WalletKind = "metamask" | "phantom" | "slush";

type WalletSetup = {
  wallet: WalletKind;
  seedPhrase: string; // 12 or 24 words; use a fresh one for Phantom and Slush
  password: string;
  version?: string; // pin an extension version (MetaMask)
  cacheDir?: string; // default ".walletwright"
};
```

## `buildCache(setup, options?)`

Imports the wallet and writes an onboarded profile to disk, then returns the profile directory. It
reuses a previously downloaded extension but re-runs onboarding on each call.

```ts
await buildCache(setup);
await buildCache(setup, { headless: true }); // onboarding only; tests still run headed
```

| Argument           | Type          | Description                               |
| ------------------ | ------------- | ----------------------------------------- |
| `setup`            | `WalletSetup` | The wallet to onboard.                    |
| `options.headless` | `boolean`     | Build headless (faster). Default `false`. |

## `createWalletFixtures(setup)`

Returns a `@playwright/test` `test` whose `context` and `page` run in a persistent Chromium with the
wallet loaded and unlocked, plus a `wallet` fixture.

```ts
const test = createWalletFixtures(setup);
const { expect } = test;

test("connect", async ({ page, wallet }) => {
  // your test
});
```

It needs a cache built for the same `setup`, via `buildCache` or the CLI.

## The `wallet` fixture

```ts
type Wallet = {
  connectToDapp: () => Promise<void>;
  confirmSignature: () => Promise<void>;
  approve: (options?: { optional?: boolean }) => Promise<void>;
  rejectConnection: () => Promise<void>;
  rejectSignature: () => Promise<void>;
  reject: (options?: { optional?: boolean }) => Promise<void>;
  network: {
    add: (config: NetworkConfig) => Promise<void>;
    switch: (chainId: number) => Promise<void>;
  };
  settings: { lock: () => Promise<void>; unlock: () => Promise<void> };
  readonly home: Page;
  readonly extensionId: string;
};

type NetworkConfig = {
  name: string;
  chainId: number;
  rpcUrl: string;
  symbol: string;
  blockExplorerUrl?: string;
};
```

| Member                | Description                                                                                                      |
| --------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `connectToDapp()`     | Approve a pending connection popup. Resolves quietly if the wallet auto-approved.                                |
| `confirmSignature()`  | Approve a pending signature popup. Throws if no popup appears.                                                   |
| `approve(options)`    | Approve any pending popup, connect or sign or transaction. `optional: true` tolerates none.                      |
| `rejectConnection()`  | Reject a pending connection popup.                                                                               |
| `rejectSignature()`   | Reject a pending signature popup.                                                                                |
| `reject(options)`     | Reject any pending popup. The counterpart of `approve`.                                                          |
| `network.add(config)` | Add a custom network from the wallet's own UI. The RPC must be reachable; the wallet validates it before saving. |
| `network.switch(id)`  | Make `chainId` the active network from the wallet's own UI, where the wallet has such a control.                 |
| `settings.lock()`     | Lock the wallet from its own UI.                                                                                 |
| `settings.unlock()`   | Unlock it again with the setup's password.                                                                       |
| `home`                | The wallet's own extension page. Named `home` because `page` means the dapp under test.                          |
| `extensionId`         | The loaded extension id.                                                                                         |

### Capabilities are per-wallet

Beyond connect and sign, a wallet only declares what has actually been driven against the real
extension, so the API never claims support it doesn't have. Calling something a wallet hasn't
declared throws rather than silently doing nothing:

```
[walletwright] Slush does not support reject()
```

Today `reject` is verified on MetaMask and Phantom, and `settings` and `network.add` on MetaMask.
Slush's onboarding can't currently be driven, so it declares none of them.

`network.switch` is declared by no wallet yet. MetaMask 13.x scopes the active chain **per dapp**
and no longer has a wallet-side network selector, so switching is dapp-initiated: the dapp calls
`wallet_addEthereumChain` (idempotent: adds when missing, switches when present) and
`wallet.approve()` drives the popup. Avoid a bare EIP-3326 `wallet_switchEthereumChain` for custom
chains; in MetaMask 13.x it hangs with no popup and no error. See the network recipe in Examples.

## `launchWallet(setup)`

The standalone launcher, without Playwright fixtures. It returns `{ context, wallet }`; close the
context yourself.

```ts
const { context, wallet } = await launchWallet(setup);
const page = await context.newPage();
// drive the page and the wallet here
await context.close();
```

## `walletwright/chain`

A local EVM node for network and transaction tests, as a separate entry point so the core
connect/sign flow doesn't depend on it. It needs the `prool` package (optional peer) and Foundry's
`anvil` binary on `PATH`.

```ts
import { createLocalChain } from "walletwright/chain";

const chain = await createLocalChain(); // { chainId: 31337, rpcUrl: "http://127.0.0.1:8545" }
// â€¦ run tests against it â€¦
await chain.stop();
```

| Option     | Default                       | Description                            |
| ---------- | ----------------------------- | -------------------------------------- |
| `chainId`  | `31337`                       | Chain id of the local node.            |
| `port`     | `8545`                        | Port for the JSON-RPC endpoint.        |
| `mnemonic` | the public Hardhat/Anvil seed | Seed for the pre-funded accounts.      |
| `binary`   | `anvil`                       | Binary to launch (resolved on `PATH`). |

The default mnemonic pre-funds `0xf39Fâ€¦92266` with 10000 ETH, the same account the MetaMask
`WalletSetup` in these docs imports, so specs can assert against it with no extra funding step.

## `wallets`

The registry of built-in `WalletDefinition`s, keyed by `WalletKind` (`"metamask"`, `"phantom"`,
`"slush"`). Useful for inspection or extension.

## `walletKindsByEcosystem(ecosystem)`

The wallet kinds that can drive a given `Ecosystem`.

```ts
walletKindsByEcosystem("evm"); // ["metamask", "phantom"]
walletKindsByEcosystem("sui"); // ["slush"]
```

## CLI

```bash
walletwright cache --setup ./wallet-setup.ts
walletwright cache --wallet metamask --seed "YOUR_SEED_HERE" --password "YOUR_PASSWORD_HERE"
```

| Flag                | Description                                       |
| ------------------- | ------------------------------------------------- |
| `--setup <file>`    | A module whose default export is a `WalletSetup`. |
| `--wallet <kind>`   | `metamask`, `phantom`, or `slush`.                |
| `--seed <phrase>`   | Seed phrase to import.                            |
| `--password <pw>`   | Wallet password.                                  |
| `--version <v>`     | Pin an extension version.                         |
| `--cache-dir <dir>` | Cache directory (default `.walletwright`).        |
| `--headless`        | Build the cache headless.                         |
